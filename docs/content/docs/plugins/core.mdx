The **core** plugin (`id: core`) supplies most **non-integration** blocks: schedules, webhooks, HTTP, conditions, database queries against Hyperdrive-backed Postgres, Workers AI text and image nodes, sub-workflows, and more. It also defines several **connection** providers used by those nodes.

## Connections

| Provider | Used for |
| --- | --- |
| **Outbound webhook** | URL plus signing secret for [Send webhook](/docs/core-nodes/send-webhook)-style calls. |
| **PostgreSQL database** | Hyperdrive binding name (and optional default schema) for [Query database](/docs/core-nodes/query-database). |
| **Workers AI** | Account ID and API token for [AI text](/docs/core-nodes/ai-text) and [AI image](/docs/core-nodes/ai-image). |
| **Custom** | Free-form base URL and API key for generic integrations. |

Exact field labels match the Connections UI for each provider.
