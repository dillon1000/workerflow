title: Concepts
description: |
  The architectural ideas behind Workerflow: durable execution, retry-safe
  effects, tracing, and the plugin standard library.
---

Workerflow is easier to reason about once you separate the system into a few core ideas:

- the graph is an authoring model
- Cloudflare Workflows is the execution substrate
- plugin steps are the integration boundary
- external effects need retry-safe handling
- traces and run history are not the same thing
